# বইয়ের OCR টেক্সট ও অনুসন্ধান-ভাণ্ডার

মূল কিতাব **এসো আরবি শিখি** (৪৪৬ পৃষ্ঠা, স্ক্যান করা) থেকে OCR করে তোলা পূর্ণ
পাঠ্য। লেখক ও প্রকাশকের অনুমতিক্রমে **কেবল শিক্ষামূলক ব্যবহারের জন্য**।

```
book-text/
  pages/p-001.json …    প্রতিটি পৃষ্ঠা — কাঁচা OCR (দুই পাসে) + সংশোধিত রূপ
  corpus.jsonl          খণ্ড/পৃষ্ঠা তথ্যসহ পরিচ্ছন্ন পাঠ
  vectors.json          TF-IDF ভেক্টর (অনুসন্ধানের জন্য)
  index.json            কোন পৃষ্ঠা কোন খণ্ডে — সারসংক্ষেপ
  ocr-corrections.json  যেসব শব্দ সংশোধন করা হয়েছে তার পূর্ণ তালিকা
```

প্রতিটি পৃষ্ঠার ফাইলে চারটি ফিল্ড:
`mixed`/`arabic` = **কাঁচা OCR, কখনো বদলানো হয় না**;
`mixedFixed`/`arabicFixed` = সংশোধিত রূপ (corpus এখান থেকেই তৈরি)।

মূল PDF ও পৃষ্ঠার ছবি (`book-scan/`) গিটে রাখা হয় না — ছবিগুলো দরকার হলে
PDF থেকে আবার তৈরি হয়ে যায়।

## প্রতিটি পৃষ্ঠায় দুইবার OCR — কেন

| ক্ষেত্র | ফিল্ড | কেন |
|---|---|---|
| বাংলা নির্দেশনা, শিরোনাম | `mixed` (`ben+ara`) | বাংলা ভালো আসে |
| আরবি শব্দ ও বাক্য | `arabic` (`ara`) | আরবিতে কম ভুল করে |

দুটোই রাখা হয়েছে, কারণ কোনটি বেশি নির্ভরযোগ্য তা পৃষ্ঠাভেদে বদলায়।

## ⚠️ নির্ভুলতা সম্পর্কে জরুরি কথা

OCR **নিখুঁত নয়**, বিশেষত **হারাকাতযুক্ত আরবিতে**। বাস্তব উদাহরণ:

| বইয়ে আছে | OCR পড়েছে |
|---|---|
| `قَدِيمَةٌ` | `تَدِيْمة` (ق → ت, হারাকাত ভুল) |
| `تَعَلَّمُوا` | `تعلمرا` (و → ر) |

লক্ষণীয়: **তাশকীল যত বেশি, OCR তত খারাপ।** ৩য় খণ্ড (কম তাশকীল) ভালো আসে,
১ম খণ্ড (পূর্ণ তাশকীল) বেশি ভুল করে।

### কী কী সংশোধন করা হয়েছে (`npm run fix:ocr`)

| সমস্যা | ফল |
|---|---|
| অদৃশ্য দিক-চিহ্ন (RLM/LRM) | **২২,৯৮০ → ০** (সম্পূর্ণ পরিষ্কার) |
| আরবি লাইনে ঢুকে পড়া ল্যাটিন/সংখ্যার আবর্জনা | মুছে ফেলা হয়েছে |
| নুক্তার ভুল (`مدرشة` → `مَدْرَسَةٌ`) | ১২৪টি শব্দ |

**শব্দ সংশোধনে ইচ্ছাকৃতভাবে খুব সতর্ক থাকা হয়েছে।** বেশি আক্রমণাত্মক নিয়মে
১৫-২০% ক্ষেত্রে *সঠিক* শব্দই নষ্ট হচ্ছিল — যেমন `لِلنَّبِيِّ` (নবীর জন্য) বদলে
`لِلْبِنْتِ` (মেয়ের জন্য) হয়ে যাচ্ছিল, অর্থই পাল্টে যেত। তাই:

- ৬ অক্ষরের কম শব্দে হাত দেওয়া হয় না (মেপে দেখা গেছে ছোট শব্দে ভুলের হার বেশি)
- আসল কিন্তু বিরল শব্দগুলোর একটি **সুরক্ষিত তালিকা** আছে
- ফলে `تَدِيْمة`-এর মতো কিছু ভুল **রয়েই গেছে** — নিরাপত্তার জন্য এটুকু মেনে নেওয়া

প্রতিটি সংশোধন `ocr-corrections.json`-এ লেখা আছে, চাইলে মিলিয়ে দেখা যাবে।

**এই টেক্সট দিয়ে সরাসরি পাঠ্যবস্তু বানানো যাবে না।** অ্যাপের আরবি
(`content/`) মানুষের যাচাই করা — সেটিই নির্ভরযোগ্য উৎস।

এই ভাণ্ডারের সঠিক ব্যবহার:
- **খুঁজে বের করা** — কোন বিষয় বইয়ের কোন পৃষ্ঠায় আছে
- **যাচাই করা** — অ্যাপের কোনো পাঠ বইয়ের কোথা থেকে এসেছে মিলিয়ে দেখা
- **ভবিষ্যতের কাজ** — নতুন পাঠ/অনুশীলন বানানোর সময় উৎস হিসেবে

সন্দেহ হলে সবসময় মূল স্ক্যান করা পৃষ্ঠাটি দেখে নাও।

## ব্যবহার

```bash
npm run ocr:book        # PDF → pages/*.json   (একবারই; থামলে আবার চালালেই চলবে)
npm run fix:ocr         # ভুল সংশোধন (কাঁচা OCR অক্ষত থাকে)
npm run build:corpus    # pages → corpus.jsonl + vectors.json
npm run search "ইদাফা"  # বইয়ে খোঁজো
```

উদাহরণ:
```bash
npm run search "সূর্য হরফ"
npm run search "الشمس" --top 5
npm run search "মাসদার" --part "২য় খণ্ড"
```

## ভেক্টর সম্পর্কে

`vectors.json`-এ **TF-IDF ভেক্টর** — সম্পূর্ণ অফলাইনে, কোনো পেইড API ছাড়া।
শব্দের মিল দেখে খোঁজে, অর্থ বোঝে না ("গরম" খুঁজলে "উষ্ণ" পাবে না)।

ভবিষ্যতে অর্থ-বোঝা (semantic) অনুসন্ধান চাইলে নিউরাল এমবেডিং লাগবে, যার জন্য
একটি এমবেডিং মডেল/API দরকার। তখনো ইনপুট এই `corpus.jsonl`-ই — কাঠামো বদলাতে
হবে না, শুধু `vectors.json` নতুন করে বানালেই হবে।
