# কন্টেন্ট সম্পাদনার নির্দেশিকা

অ্যাপের **সব পাঠ্যবস্তু** এখানে থাকে — কোনো কোড না ছুঁয়েই এগুলো এডিট করা যায়।
ভুল বানান, ভুল হারাকাত, ভুল অনুবাদ — সব এখান থেকেই ঠিক করা যাবে, ইনশাআল্লাহ।

```
content/
  book.json              পাঠের ক্রম ও খণ্ড/অধ্যায়ের ভাগ
  icons.json             শব্দের পাশে দেখানো আইকন
  lessons/01-….json …    ৩৯টি পাঠ — বইয়ের ক্রমে সাজানো
  stories/01-s1-….json … ৮টি গল্প
```

## কীভাবে বদলাবে (৩ ধাপ)

1. `content/` এর ভেতরের ফাইলটি এডিট করো (যেকোনো টেক্সট এডিটরে)
2. টার্মিনালে চালাও:  `npm run build:content`
3. `content/…` আর `js/content.generated.js` — **দুটোই** কমিট করে পুশ করো

> ⚠️ `js/content.generated.js` হাতে এডিট করো না — ধাপ ২ ওটাকে নতুন করে লেখে।

## পাঠের ফাইলে কী থাকে

```jsonc
{
  "id": 4,                    // স্থায়ী পরিচয় — কখনো বদলিয়ো না (অগ্রগতি এর সাথে বাঁধা)
  "title": "আল (ال)",         // পাঠের নাম
  "sub": "…",                 // নামের নিচের ছোট লাইন
  "icon": "🌙",
  "tip": "…",                 // ছোট টিপ (HTML চলে)
  "learn": ["…", "…"],        // নিয়ম-ব্যাখ্যা; প্রতিটি লাইন আলাদা টুকরো
  "vocab": [ {"a":"كِتابٌ","b":"একটি বই","img":"📖"} ],
  "sents": [ {"a":"هٰذا كِتابٌ","b":"ইহা একটি বই"} ],
  "qa":   [ {"q":"…","o":["…","…"],"c":0} ],   // c = সঠিক উত্তরের ক্রম (০ থেকে গোনা)
  "fill": [ {"q":"…","o":["…","…"],"c":0} ]
}
```

- `a` = আরবি, `b` = বাংলা অর্থ
- `img` **শুধু মূর্ত বস্তুতে** দাও — এটি ছবি-অনুশীলনে ব্যবহৃত হয়। বিমূর্ত শব্দ
  (ক্রিয়া, সর্বনাম, বিশেষণ) এ `img` দিলে "কোন ছবিটি সঠিক?" প্রশ্ন অর্থহীন হয়ে যায়।
  শুধু দেখানোর আইকন লাগলে `icons.json`-এ দাও।
- `learn`-এ `<b>…</b>` মোটা লেখা, `<br>` নতুন লাইন, আর
  `<span class='ar'>…</span>` আরবি লেখার জন্য।

## ভুল করলে কী হবে

`npm run build:content` নিজেই ধরে ফেলে এবং বাংলায় বলে দেয় — যেমন:

```
✖ কন্টেন্ট ত্রুটি: 05-al-definite: qa[0]-এর সঠিক উত্তরের সূচক "c" সীমার বাইরে
```

ত্রুটি থাকলে ফাইল তৈরি হয় না, তাই ভাঙা কন্টেন্ট কখনো অ্যাপে যায় না।
`npm test` দিয়েও যেকোনো সময় সব যাচাই করা যায়।

## ক্রম বা অধ্যায় বদলাতে

`book.json`-এর `order` তালিকায় ফাইলের নাম উপরে-নিচে সরালেই পাঠের ক্রম বদলাবে।
তবে খেয়াল রেখো — `sections` এর `from`/`to` সংখ্যাগুলোও তখন মেলাতে হবে, আর
পাঠের ক্রম বদলালে পুরনো ব্যবহারকারীর মুকুট ঠিক রাখতে `js/state.js`-এর
`ORDER_V` বাড়াতে হয়। এমন বদল করার আগে একবার জিজ্ঞেস করে নেওয়াই ভালো।
